# Working style

- Start with analysis and explanation by default.
- Do not run any shell commands unless I explicitly ask.
- Only read files when directly relevant to the current question.
- Do not fetch web content unless I explicitly ask.
- Do not make code changes unless I explicitly ask for a patch or implementation.
- Only change what was explicitly requested, nothing more.
- One sentence explaining why, then the change. No unrelated refactors.
- Do not proactively suggest improvements unless I ask for feedback.
- Do not write to memory unless I ask.
